# Contributing to Human Embedded Technology Wiki

Thank you for your interest in contributing to this project! This document provides guidelines for contributing to the Human Embedded Technology Wiki.

## How to Propose Changes

1. **Fork the repository** — Create your own fork of the project.
2. **Create a branch** — Make your changes in a new branch.
3. **Make your changes** — Edit or add documentation as needed.
4. **Test locally** — Run `mkdocs serve` to preview your changes.
5. **Submit a pull request** — Open a PR with a clear description of your changes.

## Style Guide for Biohacking Documentation

When contributing documentation, please follow these guidelines:

### Objective Writing

- Use clear, factual language
- Avoid sensationalism or hype
- Cite sources when referencing studies or statistics
- Be precise with technical specifications

### Safety-Focused Content

- Always include relevant safety warnings and disclaimers
- Highlight risks clearly using appropriate formatting (e.g., warning boxes)
- Never encourage unsafe practices
- Include proper aftercare and medical consultation advice where relevant

### Formatting

- Use Markdown formatting consistently
- Include a table of contents for longer documents
- Use tables for specifications and comparisons
- Include diagrams or images where helpful

## How to Submit Pull Requests

1. Ensure your changes follow the style guide above.
2. Write a clear PR title describing the change.
3. In the PR description, explain:
   - What changes you made
   - Why the changes are needed
   - Any related issues or discussions
4. Be responsive to feedback and requested changes.

## Questions?

If you have questions about contributing, feel free to open an issue for discussion.
